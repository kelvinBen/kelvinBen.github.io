Python3.7.0以上版本安装pip报错ModuleNotFoundError: No module named '_ctypes'解决方法

发表于 2019-06-07 | 分类于 疑难杂症

问题所在环境

  • CentOS 7.0
  • Python 3.8.0

错误描述

在给CentOS的Python安装pip模块的时候出现如下错误：

错误原因

Python3中有个内置模块叫ctypes，它是Python3的外部函数库模块，它提供兼容C语言的数据类型，并通过它调用Linux系统下的共享库(Shared library)，此模块需要使用CentOS7系统中外部函数库(Foreign function library)的开发链接库(头文件和链接库)。
由于在CentOS7系统中没有安装外部函数库(libffi)的开发链接库软件包，所以在安装pip的时候就报了”ModuleNotFoundError: No module named ‘_ctypes’”的错误。

解决方案

其实很简单安装一下外部函数库(libffi)就可以了，操作步骤如下：

  1. 使用yum install命令安装libffi-devel

  2. 使用make&make install 命令重新编译并安装python

  1. 最后安装pip，可以看到成功安装

步骤2报错的解决方法

  1. 在操作步骤2的时候如果出现下述错误请检查当前目录是否在python的源码目录

  2. 如果不是Python的源码目录，并且已经找不到了请按照下述步骤进行操作

  3. 然后安装pip即可安装成功。
